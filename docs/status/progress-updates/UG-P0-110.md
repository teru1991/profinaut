# UG-P0-110 Progress Update

## Entrypoints covered by centralized Policy Gate
- `POST /execution/order-intents` (`post_order_intent`)
- `POST /execution/orders/{order_id}/cancel` (`cancel_order`)

Both entrypoints now call `_enforce_policy_gate(...)` before any upstream/live client invocation.

## Precedence (safe first)
1. `SAFE_MODE` / `HALTED` hard safety flags (`SAFE_MODE_BLOCKED`)
2. Risk limits slot reserved (no execution risk limiter implemented in this service yet)
3. Live gating for GMO upstream execution
   - Live feature flag disabled (`LIVE_DISABLED`)
   - Dry-run mode (`DRY_RUN_ONLY`)
   - Backoff/degraded window (`LIVE_DEGRADED`, decision=`THROTTLE`)
4. Allow (`POLICY_ALLOW`)

## Unified gate output
`services/execution/app/policy_gate.py` returns:
- `decision`: `ALLOW | BLOCK | THROTTLE | REDUCE_ONLY | CLOSE_ONLY | FLATTEN | HALT`
- `reason_code`: machine-readable string
- `evidence`: optional structured payload
